# ✏️ Naming Strategy
다음은 네이밍 전략입니다.

## 🔍 Commit naming strategy

* 기능 구현 : "Feature : <내용>"
* 리팩토링  : "Refactoring : <내용>"
* 버그 수정 : "BugFix : <내용>"
* 기타 수정 : "Fix : <내용>"
* 문서 작성 : "Docs : <내용>"
* 기타     : "Etc : <내용>"

## 🔍 Pull & Request naming strategy

* 기능 구현 : "[Feature] <내용>"
* 버그 수정 : "[Bug] <내용>"
* 문서 수정 및 추가 : "[Docs] <내용>"
* 기타     : "[Etc] <내용>"
* CI/CD 수정 : "[DevOps] <내용>"
